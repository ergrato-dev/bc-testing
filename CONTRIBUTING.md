# Guía de Contribución

Gracias por contribuir a este bootcamp open source.

## Flujo recomendado

1. Haz un fork del repositorio.
2. Crea una rama descriptiva:

```bash
git checkout -b feat/nombre-cambio
```

3. Realiza cambios pequeños y enfocados.
4. Ejecuta validaciones del contenido (estructura semanal, enlaces y formato).
5. Crea commits claros (ideal: Conventional Commits).
6. Abre un Pull Request con contexto suficiente.

## Estándares de contenido

- Mantener nomenclatura técnica en inglés y explicaciones en español.
- Respetar la estructura semanal establecida en `bootcamp/week-XX/`.
- En `2-practicas/`: formato guiado con código comentado (sin TODOs).
- En `3-proyecto/`: usar TODOs y ejemplos de dominio genérico (Museo, Planetario, Acuario).
- Evitar cambios masivos fuera del alcance del PR.

## Convenciones de commits

Ejemplos:

- `feat: add week-19 pytest-asyncio practice`
- `fix: correct svg text overlap in week-17`
- `docs: improve rubric wording for week-18`

## Pull Requests

Incluye en la descripción:

- Qué cambió.
- Por qué era necesario.
- Cómo validar el cambio.
- Capturas si modificaste SVG o recursos visuales relevantes.

## Código de conducta

Al participar en este proyecto aceptas el [Código de Conducta](CODE_OF_CONDUCT.md).
