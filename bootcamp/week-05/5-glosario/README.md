# Glosario — Semana 05

> Primeros Tests Automatizados con Java (JUnit 5)

---

## A

**assertEquals**
Assertion para comparar valor esperado vs valor actual.

**assertThrows**
Assertion para validar que una operación lanza una excepción esperada.

---

## B

**BeforeEach**
Anotación que ejecuta setup antes de cada test.

**Build Success**
Estado de Maven cuando compilación y tests terminan correctamente.

---

## D

**DisplayName**
Anotación de JUnit 5 para mostrar nombre legible del test en reportes.

---

## E

**Error**
Fallo por excepción inesperada antes de la assertion.

---

## F

**Failure**
Fallo de assertion: el test corre pero el resultado no coincide.

---

## J

**JUnit 5**
Framework de testing para Java usado en esta semana.

---

## M

**Maven**
Herramienta de build y gestión de dependencias para Java.

**mvn test**
Comando principal para ejecutar tests en Maven.

---

## P

**Pass**
Resultado exitoso de un test.

---

## R

**Red-Green-Refactor**
Ciclo de TDD: test falla, implementación mínima, mejora.

---

## S

**Surefire**
Plugin de Maven encargado de ejecutar tests unitarios.

---

## T

**Test Method**
Método anotado con `@Test` que representa un caso de prueba.

**Test Suite**
Conjunto de tests relacionados de un mismo módulo o componente.
