# Glosario — Semana 03

> Primeros Tests Automatizados con JavaScript (Jest)

---

## A

**AAA (Arrange-Act-Assert)**
Estructura recomendada para escribir tests legibles: preparar datos, ejecutar acción, verificar resultado.

**Assertion**
Verificación que confirma que el resultado real coincide con el esperado.

---

## B

**beforeEach**
Hook de Jest que se ejecuta antes de cada test para preparar estado limpio.

**Bug**
Defecto del software que provoca comportamiento incorrecto.

---

## C

**Coverage**
Métrica que indica cuánto código o comportamiento fue ejercitado por tests.

**CLI (Command Line Interface)**
Interfaz por terminal para ejecutar comandos, por ejemplo `pnpm test`.

---

## D

**describe**
Función de Jest para agrupar tests relacionados bajo un mismo contexto.

---

## E

**expect**
Función central de Jest para declarar assertions.

**Edge Case**
Caso extremo o límite donde suelen aparecer errores.

---

## F

**Fail**
Estado de test que indica que una verificación no se cumplió.

**FIRST**
Principios de calidad para tests: Fast, Isolated, Repeatable, Self-validating, Timely.

---

## G

**Green**
Fase del ciclo red-green-refactor en la que el test pasa.

---

## H

**Happy Path**
Flujo principal exitoso del sistema.

**Hook**
Función de ciclo de vida de Jest (`beforeEach`, `afterEach`) para setup/cleanup.

---

## I

**it**
Alias de `test` en Jest para definir un caso de prueba.

---

## J

**Jest**
Framework de testing para JavaScript usado en este bootcamp.

---

## M

**Matcher**
Método que compara valor esperado vs recibido (`toBe`, `toEqual`, `toThrow`, etc.).

**module.exports**
Mecanismo de exportación CommonJS para compartir funciones entre archivos.

---

## P

**Pass**
Estado de test que indica resultado correcto según assertions.

**pnpm**
Gestor de paquetes recomendado en el bootcamp para proyectos JavaScript.

---

## R

**Red**
Fase inicial del ciclo TDD en la que el test falla intencionalmente.

**Refactor**
Mejora interna de código sin cambiar comportamiento observable.

**Runtime Error**
Error durante ejecución del test antes de llegar al assertion.

---

## S

**Sad Path**
Escenario de error o condición inválida que el sistema debe manejar.

**Suite**
Conjunto organizado de tests relacionados.

---

## T

**test**
Función de Jest que define un escenario de prueba.

**toBe**
Matcher para igualdad estricta (`===`).

**toEqual**
Matcher para igualdad profunda de objetos/arreglos.

**toThrow**
Matcher para verificar que una función lanza una excepción.

**toBeTruthy / toBeFalsy**
Matchers para verificar valores truthy/falsy.

---

## U

**Unit Test**
Prueba enfocada en una unidad pequeña de lógica (normalmente función pura).

---

## W

**Watch Mode**
Modo de Jest que vuelve a ejecutar tests al detectar cambios en archivos.
