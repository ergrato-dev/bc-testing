# Política de Seguridad

## Reporte de vulnerabilidades

Si encuentras una vulnerabilidad de seguridad, evita publicarla en issues
públicos. Reporta de forma privada a las personas mantenedoras del repositorio.

## Alcance

Este repositorio es principalmente educativo, pero igual se recomienda reportar:

- Exposición accidental de secretos o tokens.
- Dependencias con vulnerabilidades críticas conocidas.
- Scripts con comportamiento riesgoso no documentado.

## Buenas prácticas mínimas

- No subir credenciales reales.
- Usar datos ficticios en ejemplos y tests.
- Revisar secretos antes de cada commit.
